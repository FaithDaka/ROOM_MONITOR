What it Is.

The room monitor is a system that includes 2 PIR motion sensors that detect motion upto 1.5m away from their hardware(where one acts as the entrace sensor and the other the exit sensor), 3 led bulbs that represent different devices in a room that consume electricity, an arduino Uno board, a breadboard to ease connection to the board and jumper wires. Here are the steps to how it works:

  • The hardware is connected to make the system.
  • The arduino board is connected the computer and the ROOM MONITOR code uploaded on it.
  • When succesfully uploaded this should be considered;
    • The PIR sensors take about 60 seconds to analyze the infrared radiation in its environment to be able clearly distinguish human body radiation.
    • The GSM module will take a few seconds to create a gprs connection and get an ip address to be able to communicate with the ThingSpeak server.
  • Each motion detected by the Entrance sensor is stored in an integer variable that increases with each motion detected. The Exit sensor reduces the number in the motion variable each time motion is detected. This integer value is then sent to the ThingSpeak channel created for this purpose.
  • This data is graphed and can be regularly viewed to able to know the number of people within a room at a given time.

Who its For.

The project was envisioned for custodians of large buildings but can be manipulated to ease the control of rooms big and small according to the preference of the user. It is thetechnology used that matters!

ROOM MONITOR APP

The room monitor app communicates with the board to be able to remotely control the turning off and on of the lights in the system. The app prompts the user to login after which the room within which the board is in can be remotely controlled by the click of a button. The app is available on the google play store and is coming to apple store soon!
